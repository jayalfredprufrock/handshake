## Installation

```sh
npm install @jayalfredprufrock/handshake typebox
```

The Hono adapter requires Hono as a peer dependency:

```sh
npm install hono
```

## 1. Define a Contract

Each resource is its own contract — a group of endpoints relative to the group's base path. Compose those contracts into a single **api** with `createApi`; the api is the one unit your server and client both import.

```ts
// contract.ts
import { createApi, createContract } from "@jayalfredprufrock/handshake/contract";
import { Type } from "typebox";

const User = Type.Object({
  id: Type.String(),
  name: Type.String(),
  email: Type.String(),
});

// A resource contract — endpoint paths are relative to the group's base path
export const users = createContract("/users", {
  getUser: {
    method: "GET",
    path: "/:id",
    params: Type.Object({ id: Type.String() }),
    response: User,
  },

  listUsers: {
    method: "GET",
    path: "/",
    response: Type.Array(User),
  },

  createUser: {
    method: "POST",
    path: "/",
    body: Type.Object({
      name: Type.String(),
      email: Type.String(),
    }),
    response: User,
  },

  deleteUser: {
    method: "DELETE",
    path: "/:id",
    params: Type.Object({ id: Type.String() }),
    response: Type.Object({ id: Type.String() }),
  },
});

// The composed api — routes resolve to /api/users, /api/users/:id, etc.
export const api = createApi("/api", { users });
```

## 2. Create a Server

Use `buildRoutes(api, "group", handlers)` to bind handlers to a named group, then assemble the app with `createHonoApp`. Every endpoint in the group must have a handler — `buildRoutes` throws if any is missing.

```ts
// server.ts
import { serve } from "@hono/node-server";
import { buildRoutes, createHonoApp } from "@jayalfredprufrock/handshake/hono";
import { api } from "./contract";

const users = new Map<string, { id: string; name: string; email: string }>();
let nextId = 1;

const routes = buildRoutes(api, "users", {
  getUser: ({ params }) => {
    const user = users.get(params.id);
    if (!user) {
      return new Response(JSON.stringify({ error: "User not found" }), {
        status: 404,
        headers: { "content-type": "application/json" },
      });
    }
    return user;
  },
  listUsers: () => [...users.values()],
  createUser: ({ body }) => {
    const id = String(nextId++);
    const user = { id, ...body };
    users.set(id, user);
    return user;
  },
  deleteUser: ({ params }) => {
    users.delete(params.id);
    return { id: params.id };
  },
});

const app = createHonoApp({ routes: [routes] });
serve({ fetch: app.fetch, port: 3000 });
```

Handler inputs are fully typed — `params`, `query`, `headers`, and `body` are inferred from the api. Returning a value that doesn't match the response schema is a compile-time error. (To send a typed error envelope instead of a raw `Response`, declare an error on the api and `throw api.error("NOT_FOUND", ...)`.)

## 3. Create a Client

The client is generated directly from the api. Each endpoint becomes a typed method.

```ts
// client.ts
import { createFetchClient } from "@jayalfredprufrock/handshake/client";
import { api } from "./contract";

const client = createFetchClient(api, { baseUrl: "http://localhost:3000" });

const users = await client.listUsers();
const created = await client.createUser({ name: "Alice", email: "alice@example.com" });
const user = await client.getUser({ id: created.id });
await client.deleteUser({ id: created.id });
```

`fetch` defaults to `globalThis.fetch`. Method signatures adapt to the endpoint — params first, body next, and an `options` object (`query`, `headers`, `request`) last.
